# Android-timetable-app

## 概要
通学・帰宅時に「現在もしくは指定した時刻から乗れる中で最も早い列車」を即座に把握するためのAndroidアプリです。
在籍している専門学校のチーム開発演習（開発テーマは、Androidアプリなら任意）で制作しました。
JR東海の熱海駅〜浜松駅間の時刻データに基づき、ユーザーが各列車に備忘録（コメント）を残せる機能を備えています。

## 開発の背景（解決したかった課題）
- **既存アプリの課題**: 既存の乗り換え案内アプリは便利だが、「今乗れる中で一番早いのはどの列車か」だけを確認する私たち（チームメンバー）にとっては多機能すぎると感じていた。
- **個別の状況管理**: 「この電車を逃すと1限の授業に遅刻する」といった、自分にしか分からない「列車の重要度」を可視化したいと考えた。

## 主な機能
- **時刻検索**: 現在時刻から直近の列車を早い順にリスト表示。
- **My駅間登録**: アプリ起動時に、よく使う区間（例：静岡〜焼津）の検索結果を即座に表示。
- **列車コメント機能**: 検索結果の各列車に対し、「乗り遅れたら遅刻が確定」「座れる確率が高い」などのメモを保存可能。
- **上下線自動判定**: 駅コード（CA）に基づいた駅番号の差分により、上り・下りを自動で判別。

## 技術スタック
- **言語**: Java (Android SDK)
- **データベース**: SQLite
- **データ管理**: CSVインポート形式（JR東海の時刻データを独自に構造化）
- **開発ツール**: Android Studio / GitHub

## 技術的なこだわり・工夫
- **独自の判定ロジック**: 
  実在の「駅コード（CA）」を参考に、各駅に数値（0〜）を割り振りました。出発駅と到着駅の番号を比較するロジックを実装することで、ネットワーク通信を使わずにオフラインで上下線を判別する仕組みを構築しました。
- **学習の姿勢**:
初めての成果物制作、初めてのJava開発であったため、序盤はJavaのコードを生成AIに頼ることが多くなりました。ただ、仕様を理解せずに放っておくのは気持ちが悪かったため、コードを意味を公式ドキュメントや技術記事などで理解してから実装するよう意識しました。中盤から終盤にかけては、Javaの文法やオブジェクト指向の概要が掴めてくるようになったため、実装がスムーズに進みました。
